# Дебъгване на Python в VS Code

Кратка инструкция как да ползвам дебъгера в този проект.

## Какво е инсталирано

Разширенията за Python вече са във VS Code:
- `ms-python.python` (Python)
- `ms-python.debugpy` (Python Debugger — носи `debugpy` вградено)
- `ms-python.vscode-pylance`

Конфигурацията е в:
- `.vscode/launch.json` — готовите конфигурации за стартиране
- `.vscode/settings.json` — сочи към `.venv`, `src/` и `.env`

## Как да го ползвам

**1. Избери интерпретатора (еднократно)**
`Ctrl+Shift+P` → `Python: Select Interpreter` → избери `.\.venv\Scripts\python.exe`.

**2. Сложи breakpoint**
Кликни вляво от номера на реда — появява се червена точка.
Пример: `src/crew_starter/main.py` ред 30 (`result = CrewStarter()...`).

**3. Стартирай**
Отвори **Run and Debug** (`Ctrl+Shift+D`), избери конфигурация от падащото меню горе и натисни зеления ▶ (или `F5`).

Налични конфигурации:
- **crew_starter (default topic)** — `python -m crew_starter.main` с темата по подразбиране
- **crew_starter (ask for topic)** — пита те за тема при всяко стартиране
- **maads run --case titanic** — дебъгва maads CLI
- **Current file** — дебъгва отворения в момента файл

**4. Управление, когато спре на breakpoint**

| Клавиш | Действие |
|--------|----------|
| `F5` | Continue — продължи до следващия breakpoint |
| `F10` | Step Over — следващ ред (без да влиза във функции) |
| `F11` | Step Into — влез във функцията на реда |
| `Shift+F11` | Step Out — излез от текущата функция |
| `Shift+F5` | Stop |

**5. Какво гледам докато съм спрял**
- **Variables** (вляво) — стойностите на променливите в момента
- **Watch** — добавям израз (напр. `inputs`, `topic`) за следене
- **Call Stack** — веригата от извиквания
- **Debug Console** (долу) — пиша Python изрази на живо в текущия контекст
- Задържане на мишката върху променлива в кода показва стойността ѝ

## Бележка
Пакетите в проекта се управляват с **uv**, не pip. Дебъгерът няма нужда от инсталиране на пакети, но ако потрябва — ползвай `uv`.
